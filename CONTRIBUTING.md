# Contribuciones

## Problemas

Los problemas son muy valiosos para este proyecto.

- Las ideas son una valiosa fuente de contribuciones que otros pueden realizar.

- Los problemas muestran las carencias de este proyecto.

- Con una pregunta, se indica dónde los colaboradores pueden mejorar la experiencia del usuario.

Gracias por crearlos.

##  Pull Requests

Las pull requests son una excelente manera de incorporar tus ideas a este repositorio.

Al decidir si acepto una pull request, considero lo siguiente:

### ¿Se expresa claramente el propósito?

Debes indicar claramente qué problema intentas resolver con tu contribución.

Por ejemplo:

> Agregar enlace al código de conducta en README.md

No me dice nada sobre por qué lo haces. Sin embargo;

> Agregar enlace al código de conducta en README.md porque los usuarios no siempre
> consultan CONTRIBUTING.md

Me indica el problema que has encontrado, y la pull request me muestra la acción que has tomado para solucionarlo.

### ¿Es de buena calidad?

- No tiene errores ortográficos.

- Se lee bien.

### ¿Cumple con el pacto de los colaboradores?

Este repositorio tiene un [código de conducta](CODE_OF_CONDUCT.md). Eliminaré el contenido que no lo respete.
